# Positioning

## One-sentence value proposition

ZTBrowser helps platform and security teams prove, in the browser, that a web service corresponds to a verifiably attested enclave-backed workload and then explain that identity with public provenance metadata.

## Product category

Primary category:
- developer trust platform for attested web services

Secondary categories:
- browser-visible trust UX for confidential-computing-backed services
- provenance and transparency layer for enclave workloads

## What ZTBrowser is not

- not just a browser extension
- not just a Nitro deployment toolkit
- not just a facts database
- not a generic confidential-computing platform

## Core promise

ZTBrowser combines three layers:
1. cryptographic verification
2. public provenance / metadata explanation
3. operator-friendly deployment and release discipline

## Why now

- confidential-computing and attestation are becoming more visible, but still hard to explain to users
- current attestation stacks are mostly backend-oriented, not browser-visible
- AI and privacy-sensitive workflows increase the need to prove where sensitive code runs
- multi-provider confidential computing is expanding, which makes a browser-visible trust layer strategically relevant

## Strategic posture

Recommended posture for the project:
- open core platform
- public-good verifier and release/provenance format
- monetizable hosted/operator/organization-policy/compliance layers over time
