# Positioning Map

## Axis 1: who sees the trust proof?

- low: backend/operator only
- high: browser/user visible

## Axis 2: what is the primary product layer?

- low: raw infrastructure / TEE substrate
- high: user-facing trust and provenance layer

## ZTBrowser position

ZTBrowser aims for the upper-right quadrant:
- trust proof becomes visible in the browser
- provenance is tied to released workload identity
- operator tooling exists, but the differentiation is not only infrastructure management

## Comparative sketch

- CoCo / confidential-computing substrate: lower-left to middle-left
- Fortanix / Anjuna: middle-left to middle
- ZTBrowser: upper-middle to upper-right
- Micrus: product-demo path derived from ZTBrowser, but not yet canonical
